# 家系図ノート Ver.1.0 試作3

家族・親族・先祖の人物情報と関係を端末内だけで長期間記録し、SVG家系図、人物一覧、画像、印刷として確認できるローカルWebアプリです。

- 画面表示: `Ver.1.0 試作3`
- 内部バージョン: `1.0.0-prototype.3`
- IndexedDB: `family-tree-note` / バージョン `3`
- データスキーマ: `3`
- Service Workerキャッシュ: `family-tree-note-v3`
- 外部ライブラリ・外部通信・ビルド作業: なし

人物情報と写真はブラウザのIndexedDBに保存します。PNG化、印刷準備、写真圧縮、重複検出も端末内で処理し、外部へ送信しません。

## 起動方法

### index.htmlを直接開く

`index.html`をダブルクリックします。人物・関係の登録、家系図、人物一覧、JSONバックアップ、PNG保存などを利用できます。

`file://`ではブラウザの制約によりService WorkerとPWAインストールは利用できません。直接起動時はPWAマニフェストを読み込まないため、不要なCORSエラーを残しません。

### ローカルHTTPで開く（推奨）

このフォルダで次を実行します。

```powershell
python -m http.server 8000
```

ブラウザで `http://localhost:8000/` を開きます。PWAとオフラインキャッシュも利用できます。全参照を相対パスにしているため、GitHub Pagesのサブディレクトリでも公開できます。

## 試作3の追加機能

- 年月日、年月、年のみ、年頃、不明を人物ごとに登録
- 通称、別名・幼名、敬称・続柄表記、名前の補足
- 人物一覧、並び替え、絞り込み、情報不足チェック
- 姓名、生年、よみ、親・配偶者・子どもを使う重複候補検出
- 「別人として除外」と、比較確認付きの人物統合
- 基準人物を中心とした表示範囲・1〜5親等の絞り込み
- 配偶者を含める設定と世代ラベル
- PNG保存と印刷時の存命人物プライバシー設定
- A4/A3・縦横を選べるブラウザ印刷とPDF保存
- 登録時の写真縮小・圧縮と圧縮結果表示
- 試作1・試作2バックアップの後方互換復元

試作2までの人物CRUD、親族の同時追加、関係編集・解除、表示順、関係線、ドラッグ・ピンチ、基準人物、PNG、JSON、PWAは維持しています。

## 日付精度の登録方法

人物の追加・編集画面で「分かっている範囲」を選び、必要な数字だけを入力します。

| 選択 | 保存例 | 表示例 |
|---|---|---|
| 年月日 | `1950-03-12` | 1950年3月12日 |
| 年月 | `1950-03` | 1950年3月 |
| 年のみ | `1950` | 1950年 |
| 年のみ＋おおよそ | `1950` | 1950年頃 |
| 不明 | 空文字 | 生年不明 |

入力欄の右側に保存後の表示例を示します。没年月日も同じ構造です。既存の完全な`birthDate`と`deathDate`はそのまま保持し、移行時に精度`day`を補完します。

## 人物名の拡張と検索

人物には基本の姓名・旧姓・よみのほか、次を保存できます。

- `nickname`: 通称・ニックネーム
- `otherNames`: 幼名、別名、戸籍上の表記違い
- `honorific`: 敬称、続柄、家内での表記
- `nameMemo`: 漢字が不明な場合など名前の補足

ヘッダー検索と親族追加時の既存人物検索は、姓名、旧姓、よみ、通称、別名、敬称、名前の補足を共通処理で検索します。人物カードは情報過多を避け、基本の氏名、旧姓、生没年を優先します。

## 人物一覧

画面上部またはスマートフォン下部の「人物一覧」を開きます。写真またはイニシャル、氏名、旧姓、生没年、関係件数、情報不足件数を表示します。人物を選ぶと、従来の詳細サイドパネルまたは下部シートを開きます。

並び替え:

- 氏名順
- よみがな順
- 生年順
- 更新日順
- 作成日順

絞り込み:

- 存命、故人
- 生年月日不明
- 写真なし
- 親が未登録
- 関係のない人物
- 情報不足

ヘッダーの人物検索は一覧にも同時に反映されます。

## 情報不足チェック

人物一覧または設定の「情報不足を確認」を押します。次を案内として検出します。

- 姓または名が不足
- 生年月日・生年が不明
- 親が未登録
- 写真がない
- 姓または名のよみが不足
- 関係が1件もない
- 故人だが没年月日・没年がない
- 没年月日があるが故人ではない
- 没年月日が生年月日より前
- 生年月日または没年月日が未来

情報不足は保存を禁止するエラーではありません。各行を押すと該当人物の編集画面を開きます。

## 重複候補と人物統合

設定の「重複候補を確認」を開きます。姓名、よみ、姓・旧姓、生年月日または生年、共通の親、配偶者、子どもから候補スコアと理由を作ります。候補は自動統合しません。

### 別人として除外

「別人として除外」を選ぶと、2人の組み合わせを`duplicateExclusions`へ保存します。人物が削除または統合されるまで同じ候補を繰り返し表示しません。除外情報もJSONバックアップへ含めます。

### 人物を統合

1. 比較画面で残す人物IDを選びます。
2. 姓名、よみ、通称、日付、写真、メモなど、項目ごとに残す値を確認します。
3. 残す側が空欄なら、もう一方の情報を既定で選びます。
4. 確認ダイアログの後、人物と関係を統合します。

統合は`persons`、`relationships`、`settings`、`duplicateExclusions`を1つのIndexedDBトランザクションで処理します。関係先IDを付け替え、自己関係を除き、重複関係を1件へ整理し、親子循環を含む全データ検証に通った場合だけ書き込みます。失敗時はトランザクションを中断し、統合前の状態を残します。

統合の取り消し機能と自動スナップショットは未実装です。統合画面の「先にJSONを書き出す」で必ずバックアップを推奨します。

## 親族と関係の管理

人物詳細の「親族を追加」では、父または親、母または親、配偶者・パートナー、子どもを、新しい人物または登録済み人物から追加できます。新しい人物と関係は同じトランザクションで保存するため、関係保存だけが失敗して人物だけ残ることはありません。

親・配偶者・子どもの`⋮`メニューでは、相手の詳細、関係編集、表示順変更、関係解除を利用できます。関係解除で人物は削除しません。自己関係、重複、逆向き矛盾、子孫を祖先にする循環を引き続き禁止します。

## 家系図の表示範囲

家系図左下の「表示範囲」または右上の範囲表示を押します。

- 全員
- 直系のみ
- 祖先のみ
- 子孫のみ
- 祖先と子孫
- 血縁者中心
- 指定親等以内

指定親等は1〜5親等または制限なしです。法律上の厳密な親等ではなく、家系図上で関係線を何本たどるかという分かりやすい距離です。配偶者・パートナーを含めるか選べます。表示から外れた人物も削除されず、人物一覧で確認できます。

`treeViewMode`、`kinshipDepth`、`includePartners`をsettingsへ保存し、再読み込み後も維持します。

## 世代表示

表示範囲設定の「世代ラベルを表示する」を有効にします。基準人物を0世代とし、親世代、祖父母世代、子世代、孫世代などを家系図の補助ラベルで表示します。

複数の経路がある場合は自動配置が求めた世代を使います。矛盾した経路があっても、反復回数に上限を設けてアプリが停止しないようにしています。設定は`showGenerationLabels`へ保存します。

## PNG保存とプライバシー

設定の「家系図を画像保存」を押し、出力時だけ適用するプライバシー設定を選びます。

- すべて表示
- 存命人物の生年月日を隠す（初期値）
- 存命人物の写真と生年月日を隠す
- 存命人物の氏名をイニシャル表示

現在の表示範囲に含まれる家系図全体を、淡いアイボリー背景付きPNGへ変換します。通常画面とIndexedDBの人物データは変更しません。ファイル名は`家系図ノート_YYYY-MM-DD.png`です。

非常に大きい家系図では、ブラウザのCanvas上限と端末メモリを考慮して出力倍率を調整します。

## 印刷・PDF保存

設定の「印刷・PDF保存」で次を設定します。

- タイトル、補足文、作成日の有無
- A4縦、A4横、A3縦、A3横、自動
- 現在の表示範囲または全員
- 世代ラベルの有無
- PNGと同じ存命人物プライバシー設定

「印刷画面を開く」と、ブラウザの印刷ダイアログを開きます。印刷中はアプリヘッダー、操作ボタン、詳細、モーダルを隠し、家系図全体、背景、人物カード、関係線だけを用紙へ配置します。ブラウザの「PDFとして保存」を利用できます。PDFを外部サービスへ送信したり、JavaScriptライブラリで生成したりしません。

## 写真の縮小・圧縮

新しく写真を選んだ時だけ処理します。既存写真は起動時に変換せず、そのまま表示します。

- 入力上限: 30MB
- 長辺: 最大1200px
- JPEG: JPEG品質0.86を目安
- PNG/WebP: 透過を保てるWebP品質0.88を目安
- 縦横比を維持し、高品質補間で縮小
- `createImageBitmap`の`imageOrientation: from-image`を優先し、EXIF向きを可能な範囲で反映
- 圧縮後の寸法と概算容量を人物フォームに表示

画像形式をデコードできない、Canvasを使用できないなどの失敗時は、元画像を黙って保存せずエラーを表示します。

## データ構造

### persons

試作2までの項目:

`id`, `familyName`, `givenName`, `formerFamilyName`, `familyNameKana`, `givenNameKana`, `gender`, `birthDate`, `deathDate`, `isDeceased`, `birthplace`, `photo`, `memo`, `createdAt`, `updatedAt`

試作3の追加項目:

`birthDatePrecision`, `deathDatePrecision`, `birthDateApproximate`, `deathDateApproximate`, `nickname`, `otherNames`, `honorific`, `nameMemo`

precisionは`day`、`month`、`year`、`unknown`です。写真はData URLです。

### relationships

構造は試作2と同じです。

`id`, `type`, `fromPersonId`, `toPersonId`, `relationshipType`, `startDate`, `endDate`, `status`, `sortOrder`, `memo`, `createdAt`, `updatedAt`

親子は`fromPersonId`が親、`toPersonId`が子です。パートナーは方向に依存せず重複判定します。

### settings

試作2までの`focusPersonId`, `orientation`, `scale`, `schemaVersion`, `sampleInitialized`に次を追加します。

`treeViewMode`, `kinshipDepth`, `includePartners`, `showGenerationLabels`, `outputPrivacyMode`, `printSettings`

`printSettings`には用紙、タイトル、補足文、作成日、世代ラベル、プライバシー、印刷範囲を保存します。

### duplicateExclusions

重複候補から「別人」と確認した2人の組み合わせだけを保存する新規ストアです。

`id`, `personAId`, `personBId`, `createdAt`

今回`snapshots`ストアは追加していません。用途が曖昧なストアを増やさず、統合前の手動JSONバックアップを採用しています。

## IndexedDBバージョン3へのマイグレーション

DB名は`family-tree-note`のまま、バージョンを2から3へ更新します。アップグレードトランザクションで次を行います。

1. 既存personsへ日付精度・概算フラグ・名前拡張項目を安全な初期値で補完
2. 完全な`YYYY-MM-DD`は精度`day`、`YYYY-MM`は`month`、`YYYY`は`year`として推定
3. 既存settingsへ表示範囲・出力設定を補完
4. `duplicateExclusions`ストアを新規作成
5. settingsの`schemaVersion`を3へ更新

persons、relationships、settingsの既存ストア、人物ID、関係ID、写真は削除・初期化しません。既存人物があるDBには試作3サンプルを追加しません。

バージョン1から直接3へ更新する場合は、試作2の`status`と`sortOrder`補完後に試作3移行を同じアップグレード処理内で行います。

## JSONバックアップと後方互換性

試作3バックアップのトップレベル:

`appName`, `appVersion`, `schemaVersion`, `exportedAt`, `persons`, `relationships`, `settings`, `duplicateExclusions`

試作1の`data.persons` / `data.relationships` / `data.settings`形式と、試作2のトップレベル形式を引き続き復元できます。古いバックアップにない日付精度、名前項目、表示設定、除外情報は安全な初期値で補います。

復元前に人物・関係IDの重複、参照先、自己関係、重複関係、親子の逆向き矛盾、親子循環、写真形式、日付精度・概算フラグ、除外情報の参照先を検証します。検証後に4ストアを1トランザクションで置き換えます。不正なJSONでは現在データを変更しません。

## PWAとキャッシュ

Service Workerは`family-tree-note-v3`を使用します。activate時にv3以外の旧キャッシュを削除し、`index.html`、CSS、JavaScript、マニフェスト、アイコンを相対パスでキャッシュします。

## 現在の制限事項

- 非常に大規模、多重婚、複雑な養親・継親関係では線が交差する場合があります。
- 重複候補は補助判定です。同姓同名の別人や、表記が大きく異なる同一人物を完全には判定できません。
- 人物統合の取り消しと自動スナップショットはありません。事前にJSONバックアップが必要です。
- 親等は家系図上の関係距離で、民法上の厳密な親等計算ではありません。
- 印刷結果、改ページ、PDF保存方法はブラウザとプリンタードライバーにより異なります。
- 写真のEXIF向き補正とWebP対応はブラウザ機能に依存します。
- 端末間同期、共同編集、暗号化、パスコード保護は未実装です。

## プライバシー上の注意

ブラウザのサイトデータ削除、閲覧データ消去、端末故障でIndexedDBが失われる可能性があります。定期的にJSONバックアップを保存してください。共有端末では同じOSユーザーやブラウザ利用者がデータを閲覧できる可能性があります。PNG・PDFを共有する際は、出力ダイアログの存命人物プライバシー設定を確認してください。
