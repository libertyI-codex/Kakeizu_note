# 家系図ノート Ver.1.0 試作4 修正1

内部バージョン: `1.0.0-prototype.4-fix.1`  
IndexedDB: `family-tree-note` / バージョン `4`  
データスキーマ: `4`

家族・親族・先祖の人物情報、関係、出来事、根拠資料を端末内だけで管理するローカルWebアプリです。外部API、CDN、クラウド、アクセス解析は使用せず、人物写真と資料添付もIndexedDBへ保存します。

## 起動方法

### index.htmlを直接開く

[`index.html`](./index.html)をダブルクリックして開きます。通常機能とIndexedDB保存を利用できます。`file://`ではブラウザの制約によりService WorkerとPWAキャッシュは利用しません。

### ローカルHTTPで起動する（推奨）

このフォルダで次を実行し、`http://localhost:8000/`を開きます。

```powershell
python -m http.server 8000
```

HTTP起動ではPWAとオフラインキャッシュを利用できます。GitHub Pagesへ配置する場合も、すべて相対パスのためサブディレクトリで動作します。

初回起動時だけ、表示確認用のサンプル家族を登録します。既に人物データがある場合や、試作1〜3から移行した場合はサンプルを追加・上書きしません。

## 画面と基本操作

### 家系図

ヘッダーの家系図名を押すと家系図管理を開きます。家系図カードから切り替え、名称・説明・表紙色の編集、複製、アーカイブ、削除ができます。

削除時は家系図名の再入力が必要です。人物、関係、出来事、資料、引用、添付をまとめて削除するため、削除前に手動扱いのスナップショットを作成します。最後の1件は削除できません。

家系図はドラッグ移動、ホイールまたはボタンで拡大・縮小できます。人物カードを押すと、PCでは右サイドパネル、スマートフォンでは下から詳細シートを開きます。

### 人物の別家系図へのコピー・移動

人物詳細の「別の家系図へ」から操作します。

- 人物だけ、またはつながっている関連人物を含める
- コピー、または移動
- 対象に含まれない相手との関係は作成しない
- コピー時は人物、写真、対象の出来事、資料、引用、添付を新しいIDへ安全に複製する

家系図をまたぐ関係は登録できません。移動元に残す必要がある共有資料は自動削除しません。

### 資料と添付ファイル

「資料」画面で追加・編集・削除・検索を行います。種類、信頼性で絞り込み、更新日、資料名、取得日で並び替えできます。

資料の種類:

- 戸籍・除籍、証明書、写真、聞き取り、手紙、日記
- 墓碑、新聞、書籍、Webサイト、本人・家族の記憶、その他

信頼性は「信頼性が高い」「参考情報」「未確認」「不明」です。整理の補助であり、アプリが事実を自動判定するものではありません。

添付の制限:

- 対応形式: JPEG、PNG、WebP、PDF
- 1ファイル: 20MBまで
- 1資料の合計: 50MBまで
- 10MBを超える選択ファイルには大容量表示
- 画像にはサムネイルを作成
- 「軽量化して保存」では長辺2200pxを超える資料画像を縮小
- PDFはアイコン表示し、元ファイルを開く・端末へ保存できる

人物写真は従来どおり、EXIF向きを可能な範囲で考慮し、縦横比を保ったまま長辺1200px以内へ調整します。既存写真は起動時に変換しません。

### 引用と確認状態

人物詳細の各項目にある資料アイコン、人物年表、関係メニュー、出来事から資料を引用できます。同じ情報へ複数資料を登録できます。

引用には対象項目、頁・箇所、引用文、メモ、確認状態を保存します。引用解除は`citations`だけを削除し、資料や添付は削除しません。資料削除時は引用件数と添付件数を確認し、引用と出来事内の資料参照を安全に解除します。

人物・関係・出来事には次の確認状態を設定できます。

- `confirmed`: 確認済み
- `probable`: 可能性が高い
- `unconfirmed`: 未確認
- `disputed`: 情報が食い違っている

未確認・食い違いは、小さな記号や線種でも示し、色だけには依存しません。

### 人物年表と家族年表

人物詳細の「年表・出典」では、その人物の出来事と年齢を確認できます。「家族年表」では現在の家系図全体を年代順に表示します。

絞り込み:

- 人物
- 出来事の種類
- 年代
- 確認状態
- 出典の有無

出来事の日付は、年月日、年月、年のみ、年頃、不明に対応します。生年月日が分かる範囲で年齢を計算し、精度が年だけの場合は概算表示にします。未来の出来事や没年月日後の出来事は控えめに案内します。

出来事の種類は、出生、死亡、結婚、離婚、養子縁組、居住、学歴、職歴、兵役、移住、病歴、受賞、埋葬、その他です。センシティブな出来事には`isSensitive`を保存できます。

### スナップショットと元に戻す

自動スナップショットを次の直前に作成します。

- 人物統合
- 人物削除
- 関係解除
- 出来事削除
- 資料削除
- JSON復元
- GEDCOM読み込み
- 大量削除・サンプル再登録

家系図削除前と利用者が作成したものは手動スナップショットとして扱います。自動スナップショットは家系図ごとに直近10件を保持し、手動スナップショットは自動削除しません。復元前にも現在状態を自動保存します。

主要操作の直後は画面下部の「元に戻す」を10秒間利用できます。再起動後は「スナップショット」一覧から復元します。

スナップショットに含むもの:

- 家系図、人物、人物写真、関係、表示順
- 出来事、資料メタデータ、引用
- 重複候補除外、家系図固有設定
- 添付ファイルの参照情報

資料添付のBlob本体は容量増加を避けるため複製しません。資料・家系図を削除した後にスナップショットを復元しても、削除済み添付本体は戻りません。完全な保全にはJSONバックアップを使用してください。

### 統計

現在の家系図について、人物数、存命・故人数、世代数、生年範囲、関係数、出典付き人物、未確認情報、写真登録率、生年月日登録率、未接続人物、重複候補、資料数、出来事件数を表示します。カードから関連画面へ移動できます。

## IndexedDB バージョン4

DB名は`family-tree-note`です。

| ストア | 主な用途 | 主なインデックス |
|---|---|---|
| `trees` | 家系図名、説明、基準人物、色、アーカイブ状態 | なし |
| `persons` | 人物、写真、日付精度、名前拡張、確認状態、`treeId` | `familyName`, `givenName`, `updatedAt`, `treeId` |
| `relationships` | 親子・パートナー、状態、表示順、確認状態、`treeId` | `type`, `fromPersonId`, `toPersonId`, `treeId` |
| `settings` | グローバル設定と家系図固有設定 | なし |
| `duplicateExclusions` | 「別人として除外」した重複候補 | `treeId` |
| `events` | 複数人物に紐づく出来事・年表 | `treeId` |
| `sources` | 資料の書誌・取得情報 | `treeId` |
| `citations` | 資料と人物・関係・出来事の結び付け | `treeId`, `sourceId`, `targetKey` |
| `attachments` | 資料の画像・PDF Blobとサムネイル | `treeId`, `sourceId` |
| `snapshots` | 添付本体を除く復元用履歴 | `treeId` |

`settings`のキー:

- `app`: 現在の家系図ID、スキーマ、移行済み状態などのグローバル設定
- `tree:<treeId>`: 基準人物、倍率、表示範囲、世代ラベル、出力設定などの家系図固有設定

### 試作3からのマイグレーション

IndexedDBの`onupgradeneeded`トランザクション内で、次を一度だけ行います。

1. 新ストアと必要なインデックスを作成
2. `家族の家系図`を1件だけ作成
3. 既存`persons`、`relationships`、`duplicateExclusions`へ同じ`treeId`を補完
4. 既存人物・関係へ安全な確認状態初期値を補完
5. 既存`settings`を維持し、グローバル設定と家系図固有設定へ関連付け
6. `migrationV4Complete`とスキーマ4を記録

既存の人物ID、関係ID、写真、日付精度、関係状態、表示順、表示条件、重複候補除外情報は変更しません。移行トランザクションが失敗した場合、IndexedDBがアップグレード全体をロールバックします。DBを削除・初期化する処理はありません。

## JSONバックアップ

### 全体バックアップ

「全体JSONを書き出す」は次を含みます。

- `appName`, `appVersion`, `schemaVersion`, `exportedAt`, `scope`
- `trees`, `persons`, `relationships`, `events`, `sources`, `citations`
- 写真を含む人物情報
- Data URL化した`attachments`
- 家系図ごとの`settings`
- `duplicateExclusions`

スナップショット履歴は通常バックアップへ含めません。

### 家系図単体バックアップ

設定の「この家系図をJSON保存」は現在の家系図と、その人物、関係、出来事、資料、引用、添付、設定だけを書き出します。

### 復元方法

- 新しい家系図として復元: IDを安全に付け替え、既存データを上書きしない
- 同じIDが存在しない場合のみ追加: 元IDを維持し、1件でも衝突する場合は全体を中止
- 現在の家系図を置き換える: 家系図単体バックアップだけを受け付け、事前スナップショット後に置換

復元は人物ID重複、存在しない参照、自己関係、親子循環、引用先、資料参照、出来事の人物参照、添付上限を事前検証します。書き込みは1トランザクションで行い、途中失敗時は中途半端なデータを残しません。

試作1の`data`内包形式と、試作2・試作3のトップレベル形式を読み込めます。`trees`や`treeId`がない旧バックアップは、新しい家系図を作成して不足項目を安全に補完します。

## GEDCOM 5.5.1相当

### 書き出し

設定の「GEDCOM入出力」から、現在の家系図をUTF-8で保存します。書き出し前に対象人物数・関係数と個人情報の注意を表示します。

主な出力タグ:

`HEAD`, `SOUR`, `GEDC`, `CHAR`, `LANG`, `INDI`, `FAM`, `NAME`, `GIVN`, `SURN`, `SEX`, `BIRT`, `DEAT`, `DATE`, `PLAC`, `FAMS`, `FAMC`, `HUSB`, `WIFE`, `CHIL`, `MARR`, `DIV`, `NOTE`, `TITL`, `AUTH`, `PUBL`, `PAGE`, `QUAY`

一部の出来事は`ADOP`, `RESI`, `EDUC`, `OCCU`, `IMMI`, `BURI`, `EVEN`でも出力します。アプリ固有の情報は`NOTE`または`_FTN_`で始まる独自タグへ補足します。

### 読み込み

ファイルを解析して、家系図名、人物数、家族・関係数、出来事件数、警告、未対応タグ、文字コード表記を確認してから、新しい家系図へ取り込みます。既存家系図を直接上書きしません。

自己関係、循環、重複関係、存在しない参照を拒否します。想定外のタグは可能な範囲で警告として数え、処理全体は停止しません。HEADがない、または人物がない壊れたファイルは安全に拒否します。

GEDCOMにはソフトごとの実装差があります。すべての独自タグ、日付表現、同性・複数パートナー表現、資料構造を完全再現することは保証しません。読み込み後に人物・関係・年表を確認してください。

## プライバシーと出力

- 入力データと添付を外部へ送信しません。
- PNGと印刷では、存命人物の日付、写真、氏名を出力時だけ保護できます。
- JSONとGEDCOMには家系情報そのものが含まれます。
- 出力ファイルをSNSや公開サイトへ直接アップロードせず、安全な場所へ保管してください。
- 暗号化バックアップは未実装です。
- ブラウザの「サイトデータ削除」を行うとIndexedDBも削除される場合があります。

## 現在の制限事項

- GEDCOMは5.5.1相当の初期実装で、すべてのソフトとの完全互換ではありません。
- 独自GEDCOMタグや複雑な資料・引用構造を完全には再現できません。
- 添付保存容量は端末・ブラウザ・空き容量に依存します。
- 自動スナップショットは完全バックアップの代替ではありません。
- スナップショットは資料添付のBlob本体を複製しません。
- 人物移動時、移動対象外の人物との関係は引き継ぎません。
- 大規模・複雑な近親関係では、家系図線が交差する場合があります。
- クラウド同期、暗号化バックアップ、複数人の同時編集は未実装です。
- iOS Safariのプライベートブラウズや保存容量不足ではIndexedDB保存に失敗する場合があります。
